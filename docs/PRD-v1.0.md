# Moboost AI AdTech 平台产品需求文档 (PRD v1.0)

> 版本: v1.0 | 日期: 2026-03-06 | 负责人: xu
> 核心架构: OpenClaw (中枢大脑) + n8n (自动化编排) + 飞书 (人机协作)

## 1. 产品愿景与战略规划

### 1.1 商业目标
通过解决出海广告主在竞品调研、素材本地化、合规风控上的核心痛点，利用 AI Agent 提升人效上限，最终通过获取客户 SDK 数据形成交付增长飞轮，实现从工具到流量平台的跃迁。

### 1.2 三阶段发展路径
- **第一阶段（当前）**：重服务，不碰代码。通过 AI 生成报告和素材，人工审核确保质量，验证客户兴趣。
- **第二阶段**：轻 SaaS，快速搭建 Demo。以 OpenClaw 为核心搭建 Web UI 门面，依赖数据源清洗。
- **第三阶段**：走流量，形成产品化能力。推广 SDK 授权，通过获取的数据让投放更准，形成业务飞轮。

## 2. 整体页面设计框架 (The Facade)

### 2.1 品牌视觉逻辑
采用 Moboost 统一视觉系统，保持现有页面风格。

### 2.2 核心导航结构
- **Home (控制台)**：展示 Agent 工作状态及各模块任务进度
- **Dashboard**：展示核心重点数据情况，聚合三个页面最重要的功能
- **Intelligence Radar (情报雷达)**：多级监控体系
- **Localization Engine (本地化工厂)**：素材加工工作台
- **Risk Scanner (合规扫描)**：上架前风险预警

## 3. 功能模块详细说明

### 3.1 竞品情报雷达 (Intelligence Radar)

**核心目标：** 解决数据源（如 FortyTwoMatters）数据脏、颗粒度粗的痛点。

#### Level 2：市场监控大盘
- **监控表 (Table View)**：完整展示 4 大核心维度：
  - 流量规模 (MAU/下载)
  - 用户质量 (留存/时长)
  - 投放强度 (新广告数)
  - 商业化能力 (ARPU/内购)
- **异动提醒**：实时标注竞品重大动作（如支付变更、新包上架）
- **推荐监控**：AI 自动发现潜在对手

#### Level 3：单产品深度剖析 (以 Thrillzz 为例)
- **核心走势图**：用户增长与广告消耗的正相关性分析
- **留存流失图**：对比行业基准线
- **行为时间轴**：记录近 30 天所有策略变动（支付网关、活动、合规路径）
- **专家结论**：提供可执行的投放建议

### 3.2 素材本地化工厂 (Localization Engine)

**核心目标：** 建立"AI 生成 + 人工审核"的轮值体系，提升广告效果上限。

- **素材流水线**：实时展示任务状态（解析中、审核中、已就绪）
- **本地化对比**：原视频与本地化视频（翻译、配音、视觉改写）同屏对比
- **文化适配引擎**：OpenClaw 提供的当地热语、禁忌与趋势建议
- **审核席入口**：对接飞书审批，人工一键通过或驳回

### 3.3 合规扫描仪 (Risk Scanner)

**核心目标：** 辅助产品非核心卖点，解决"投不出去"的痛点。

- **风险评分**：基于 Meta/TK 政策库的预审分值
- **改写建议**：AI 标注违规片段并给出具体修改方案
- **Cloaking 部署**：对接第三方 API，配置 Safe Page 与 Target Page 跳转

## 4. 数据与技术底层要求

### 4.1 数据源对接
- **FortyTwoMatters**：需进一步获取详细数据样本，特别是 ARPU、留存和原始数据颗粒度
- **数据清洗**：由数据组（筹建中）负责数据治理，与研发分离

### 4.2 AI 设施与资源
- **OpenRouter**：统一企业账号，初期预充 $2000
- **n8n / OpenClaw**：初期用于快速搭建工作流逻辑，验证需求
